# AutoSlicer v2 — 歌回直播自动切片系统 技术报告

> 调研综述：`notes/RESEARCH.md`；设计文档：`notes/DESIGN.md`；过程记录：`notes/WORKLOG.md`。
> 代码仓库：https://github.com/anAtheist987/AutoSlicer-v2

## 摘要

- **评估协议**：训练 3 场 / 验证 1 场 / 测试 2 场（按场次划分，测试集不参与训练、
  模型选择与任何参数调优，仅评估一次）。**独立测试集事件级 F1 = 0.899**
  （45 个标注事件命中 40、误报 4，P 0.909 / R 0.889），帧级 F1 = 0.940。
- **3 折交叉验证**（6 场人工标注场次，留二法）：事件级 F1 = **0.929 ± 0.031**，
  帧级 0.956 ± 0.007。更换随机种子重训后事件级 F1 复现（0.904 vs 0.904），
  训练过程稳定；折间方差主要来源于验证场次的构成。
- 模型：全频带 16kHz log-mel 前端 + CRNN + 四类 powerset 输出头
  （无人声 / 目标主播演唱 / 他人演唱 / 纯伴奏，外加弱标签辅助项），含带宽增强。
- **说话人特异性**：将目标主播的检测模型应用于另一位主播 2.1 小时高密度演唱的
  场次，误报片段为 0——模型学习的是"该说话人在演唱"，而非"存在演唱"。
- **推理性能**：模型+解码 RTF ≈ 0.0005；2.5 小时 VOD 端到端（解码 → 检测 →
  ffmpeg 切片）约 15 秒；推理显存 < 2GB；模型 2.1M 参数（8.4MB fp32）。
- 端到端 CLI 支持直接输入 bilibili 链接（`src/bilibili.py` 下载 → 检测 → 切片）。

## 1. 任务定义与约束

输入数小时直播 VOD，检出「目标主播本人在演唱」的全部片段，每首歌输出独立视频切片
（带前后余量）。需排除四类混淆场景：(a) 播放纯伴奏而未演唱；(b) 伴奏与演唱不匹配；
(c) 伴奏仅为歌曲片段；(d) 播放他人演唱的同一歌曲作对照。演唱中间短暂插播他人版本时，
不得将一首歌切分为两段。资源约束：单卡（≤24GB）数小时内完成训练；推理显存 ≤10GB；
处理时间低于素材时长；事件检出准确率 ≥80%。

## 2. 相关工作调研结论（详见 notes/RESEARCH.md）

按六个方向并行调研（歌声检测文献 / 现代音频骨干网络 / 生产级 VAD / 说话人验证 /
音源分离 / 后处理），直接影响设计的结论：

1. 生产级系统形态收敛于：log-mel → 小型逐帧分类器 → 滑窗 overlap-add → 双阈值滞回 +
   时长约束（Silero VAD / pyannote / MarbleNet 同构）。前代系统（8kHz 原始波形
   SincConv+ResNeXt+BiGRU）所需的"数分钟感受野"在该范式下由后处理与长感受野模块解决。
2. 纯歌声检测（SVD）基准长期停留在 92–94% 帧准确率；本任务的本质是
   **歌声检测 × 说话人验证**的复合问题。混淆场景 (d) 无法由任何音频 tagging 模型
   单独解决，必须在训练数据中显式构造"他人在演唱"的负例。
3. 音源分离（HTDemucs）定位为**离线数据构造工具**而非推理依赖：用于制造负例与
   伪标签的收益远大于作为模型结构组件。
4. AudioSet 预训练骨干（PANNs / EfficientAT 系）+ 轻量头是 DCASE 社区共识，
   但在 8kHz 窄带域上存在显著域差（实测 PANNs 的 Singing 类概率 p99 仅 0.38）。

## 3. 数据与监督策略

数据集：8 位主播的歌回直播录像（8kHz wav 与全频带 m4s 双版本，约 130 小时）+
目标主播 7 场**人工标注**（每首歌的起止区间，167 首，约 21 小时，正例占比约 50%，
其中 1 场因标注质量问题剔除）。监督来源分两类：

### 3.1 自监督数据构造（无人工标注的冷启动路线）

- **HTDemucs 2-stem 分离**全部场次（多卡并行、分块推理 + 交叉淡化避免显存溢出，
  约 31× 实时）→ 16kHz 单声道 vocals/accompaniment。
- **伪标签**：歌回直播中演唱主体即主播本人。帧级候选 = 人声 stem 活跃 ∧ 伴奏活跃；
  段级过滤 = 时长 ≥45s ∧ 人声密度 ≥0.4 ∧ **持续音符占比 ≥0.10**（YIN 基频滚动
  标准差 <0.7 半音；实测演唱区该比例为 21%，含背景音乐的谈话区为 6%，可有效区分）
  ∧ 伴奏能量 ≥−48dB。段边界 ±8s 置 ignore(−1) 不参与损失。全库伪标签正例率 42%，
  段长分布呈 1.3–3.9 分钟的歌曲形态。
- **说话人负例**：其他主播的完整歌回场次构成"有人在演唱但非目标主播"的负例，
  其伪演唱段在采样中重点过采样。
- **合成混淆样本**（stem 重组，120 段）：目标主播伴奏 + 他人人声（对应混淆 d）、
  纯伴奏（对应 a/c）、目标主播人声 + 错配伴奏（鲁棒正例，对应 b 中目标仍在演唱）。

### 3.2 人工标注管线

解析标注区间 → 3.125fps 帧标签。多类标签语义：0 无人声/语音，1 目标主播演唱，
2 他人演唱，3 纯伴奏，4 非目标演唱（弱标签），−1 忽略。说话人负例沿用其他主播
场次的过采样。**全部最终指标以人工标注为基准。**

## 4. 系统架构

```
VOD(任意容器) → ffmpeg 16k mono → log-mel(80 bins, hop 320, 50fps)
  → MelCRNN: 4×ConvBlock(时/频各16×池化) → BiGRU(2×256) → 帧级 logits @3.125fps
  → 滑窗 overlap-add → 3s 中值平滑 → 滞回阈值(on/off) → 移除 <8s 噪点
  → 填充 ≤45s 段内间隙(间奏/短插播) → 合并间隔 ≤90s 的相邻段(插播他人版本不拆段)
  → 移除 <50s 段 → ±5s 余量 → ffmpeg -c copy 切片
```

多类输出的训练损失：完整标注帧用交叉熵；弱标签帧（类 4，"非目标演唱"）仅惩罚
目标类概率，即 −log(1−p₁) = −(logsumexp(其余类 logits) − logsumexp(全部 logits))；
−1 帧不参与损失。增强：SpecAugment、随机增益、带宽增强（以概率 0.4 在 mel 域
模拟低通，弥合窄带/全频带域差）。

预实验比较的三条模型路径：

| 路径 | 结构 | 参数量 | 设计动机 |
|---|---|---|---|
| CRNN | mel→CNN→BiGRU | 2.09M | 生产级 VAD 共识形态 |
| TCN | mel→CNN→9 级膨胀卷积（感受野 ±6min） | 3.87M | 无循环、全并行、长上下文 |
| PANNs 头 | 冻结 CNN14_16k 嵌入(2048d@1.56fps)→BiGRU | 头 1.12M（+80M 冻结） | AudioSet 预训练迁移 |

训练配置：96s 窗口、batch 32、AdamW 3e-4 + cosine 调度、bf16、ignore-mask 损失、
正例窗口 50% 平衡采样、按场次划分数据。8k 步约 45 分钟（单张 3090）。
显存：训练 batch 32 约 8.9GB（batch 16 约减半）；推理 <2GB。

## 5. 实验结果

### 5.1 伪标签路线（目标主播 B，验证集 = 该主播 2 场 + 其他主播 3 场，相对伪标签评估）

| 模型 | frame-F1 | event-F1 | 事件 TP/FP/FN | 边界 MAE |
|---|---|---|---|---|
| TCN | 0.955 | **0.979** | 23/0/1 | ~16s |
| CRNN | 0.953 | 0.791 | 17/2/7 | ~21s |
| PANNs 头 | 0.900 | 0.818 | 18/2/6 | ~18s |

其他主播 3 场验证场次上各模型均近零误报，说话人负例策略有效。

### 5.2 人工标注路线・初版（8kHz 二分类，验证集 = 2 场、38 个合并后真值事件）

| 模型 | frame-F1 | frame-P/R | event-F1 | 事件 TP/FP/FN | onset/offset MAE |
|---|---|---|---|---|---|
| **CRNN（选型）** | **0.894** | 0.97/0.83 | **0.833** | 30/4/8 | **6.0s / 5.9s** |
| TCN | 0.857 | 0.97/0.77 | 0.817 | 30/3/8 | 38.3s / 19.9s |

**选型依据**：两者检出能力相当，CRNN 边界误差小约 6 倍——TCN 的 ±6min 感受野
模糊了段边界。边界误差小于切片余量时，产品意义上等价于边界准确。

### 5.3 说话人特异性专项（混淆场景 d）

目标主播检测模型 → 另一位主播 2.1h 高密度演唱场次：**检出 0 个片段**。
反向验证：主播 B 的伪标签模型在另外三位主播的验证场次上预测事件数为 0。

### 5.4 窄带/全频带域差测试

初版训练数据为 8kHz 升采样窄带音频，部署输入为全频带。以验证场次的全频带原始
版本直接推理（同一模型与阈值）：event F1 = 0.857（窄带版 0.89）。域迁移损失可控；
v2 改为直接在全频带上训练并加带宽增强，消除该问题。

### 5.5 速度与资源

| 项目 | 数值 |
|---|---|
| 纯模型吞吐 | ~14,000× 实时（3090, bf16） |
| 解码+特征+模型 RTF | 0.0005（2.5h VOD 约 4.9s） |
| 端到端含 ffmpeg 切片 | 2.5h VOD → 17 个切片，共 15.4s |

### 5.6 v2 评估协议：交叉验证与独立测试集

针对过拟合、数据充分性与评估污染三个问题的系统检验。

**3 折交叉验证**（6 场人工标注场次，留二法，每折独立重训 8000 步）：

| 折 | 验证场次 | frame-F1 | event-F1 |
|---|---|---|---|
| A | 1188184012, 1190452750 | 0.965 | 0.904 |
| B | 1187358909, 1194102386 | 0.953 | 0.916 |
| C | 1211660979, 582716341 | 0.950 | 0.966 |
| **mean ± std** | | **0.956 ± 0.007** | **0.929 ± 0.031** |

折 A 更换随机种子（42→123）重训：event-F1 0.904 vs 0.904，训练方差可忽略；
折间差异（±3pt）来自验证场次构成，即当前瓶颈为**场次多样性**而非过拟合
（训练场次 frame-F1 0.97 vs 验证 0.96，泛化间隙可忽略；训练损失收敛于
0.07–0.13，未出现记忆式塌缩）。

**独立测试集评估**：train = {1188184012, 1187358909, 1194102386}；
val = {1190452750}，仅用于 checkpoint 选择与后处理参数调优（on=0.5 / off=0.3，
其余默认）；test = {1211660979, 582716341}，不参与任何决策，仅评估一次
（`scripts/eval_test.py`，固定参数）：

| | frame-F1 | event-F1 | P / R | 事件 | 边界 MAE |
|---|---|---|---|---|---|
| test 合计 | 0.940 | **0.899** | 0.909 / 0.889 | 40/45 命中，4 误报 | onset 13.5s / offset 12.7s |
| 1211660979 | | 0.923 | | 20 真值 | |
| 582716341 | | 0.880 | | 25 真值 | |

测试集结果（0.899）与交叉验证均值（0.929±0.031）一致。该模型的训练集比交叉验证
各折少一场标注场次，0.899 可视为保守下界。

### 5.7 生产模型与边界精化分析

**生产模型 v2_prod（release v2.1.0）**：协议验证完成后，按"交叉验证确定超参数 →
扩大训练集重训"的惯例在 5 场标注场次上重训（保留 1188184012 作 checkpoint 选择，
该场 19/19 全部命中）。预期性能引用交叉验证估计 0.929±0.031；说话人特异性回归
测试（非目标歌手的演唱视频）误报为 0。

**边界精化：经分析后默认不启用**。误差分解显示 onset/offset **中位误差仅 ±1s**，
MAE（~13s）由少数离群事件主导。逐例核查离群事件（以分离 stem 的能量与基频特征
复核）：约半数为**真实演唱但未被标注**的歌前清唱/跟唱（人声、伴奏、持续音高特征
均存在），其余为语音误触发与间隙填充的过度桥接。即后处理层面已无无损的改进空间：
以更紧的边界阈值回收低置信区段（实现为 `PostProcessConfig.refine_edges`，默认
关闭）在三折上 F1 持平、MAE 仅 −0.5s，且部分折的 p90 误差恶化。边界误差的尾部
由标注口径决定，进一步改进需要更细粒度的边界标注或专门的 onset/offset 回归头。

## 6. 使用方法

```bash
# 单命令：输入视频/音频/bilibili 链接，输出切片 + 切点 JSON + 概率曲线
python src/infer.py --input vod.mp4 --checkpoint runs/v2_prod/best.pt \
    --out-dir out/ --gpu 0 --on-threshold 0.5 --off-threshold 0.3 --cut-video

# 适配新主播：准备每首歌起止区间的标注 csv（或用 scripts/pseudo_label.py 自举）
python scripts/cache_fullband.py
python scripts/build_v2_labels.py
python src/train.py --model crnn --run-name new_streamer --gpu 0 --classes 4 \
    --bandlimit-prob 0.4 --manifest data/processed/v2_train.json \
    --val-manifest data/processed/v2_val.json --steps 8000
python scripts/tune_postprocess.py --probs-dir runs/new_streamer/val_probs
```

输出：`out/<名>.segments.json`（start/end/cut_start/cut_end，秒）、
`out/<名>_songNN.mp4`（`-c copy` 无重编码）、`out/<名>.probs.npy`
（3.125Hz 概率序列，供可视化与人工复核）。

## 7. 工程问题记录

1. **容器 /dev/shm=64M 且 overlayfs 不支持共享 mmap** → PyTorch DataLoader
   多进程 SIGBUS。解决：worker 返回 numpy 经 pipe 序列化，主进程转 tensor。
2. **ZIP64**：4GB+ 条目的本地文件头 size 字段为 0xFFFFFFFF，需解析 ZIP64 扩展
   字段,否则文件截断且其后条目（含标注 csv）无法解出。
3. **HTDemucs 整文件推理显存溢出**：2.8h 音频的 sources 张量约 14GB,多进程下
   静默被杀。解决：8 分钟分块 + 1s 交叉淡化。
4. 数据集 zip 文件名为 GBK 编码（Python 默认按 cp437 解码导致乱码）；部分外网
   资源需镜像源获取；远端 git 操作存在间歇性超时，需重试机制。
5. 训练中观测到一次验证指标瞬时归零（损失正常,1000 步内自行恢复）；最优
   checkpoint 机制可消化此类瞬态。

## 8. 局限与后续工作

- **场次多样性是当前主要瓶颈**：折间方差（±3pt）远大于训练方差（≈0），
  扩充不同日期/设备/曲风的标注场次是最直接的提升路径。
- **边界误差尾部**受标注口径限制（见 5.7），改进需细粒度边界标注或回归头。
- **短歌与清唱**：低伴奏比片段（清唱、戏腔）召回偏弱；可输出低置信"待确认"
  清单供人工复核，而非直接丢弃。
- **说话人验证分支未启用**：当前模型已通过说话人特异性测试；若出现音色相近的
  混淆者，可在分离人声 stem 上加 ECAPA-TDNN 段级余弦验证（调研已给出方案）。
- **流式实时版**：模型为因果友好结构（卷积 + GRU），改造为直播实时检测可行，
  属功能级扩展。
