# AutoSlicer v2 设计文档

> 2026-06-12 03:00。依据：notes/RESEARCH.md（调研结论）+ 数据集实际情况（见下）。

## 1. 数据现实与监督策略

数据集（仍在传输，已提取 44 文件）：`data_8k/歌回【东雪莲|小柔channel|月隐空夜|猫雷】/*.wav`
= 4 位主播的整场歌回直播（8kHz 单声道，每场 0.1~2.8h，已到手 ~90h）；`convert/` 为无关样例。
**人工标注文件尚未到达**（zip 尾部，传输近乎停滞 ~6KB/s）。PANNs 在 8kHz 窄带音频上
Singing 检测严重退化（p99=0.38），弱标签路线不可靠。

→ **主路线：自监督数据工厂**（标注若到达则用作金标验证/微调）：

1. **HTDemucs 2-stem 分离**全部 VOD（3×3090 并行，34×RT）→ 16k 单声道 vocals/accomp stem。
2. **伪标签**（scripts/pseudo_label.py）：歌回直播中唱歌主体=主播本人。
   帧级（0.32s）：人声 stem 活跃 ∧ 伴奏 stem 活跃 → 候选；段级过滤：
   时长≥45s、人声密度≥0.4、持续音符占比≥0.10（YIN F0 滚动标准差<0.7 半音）、伴奏≥-48dB。
   实测区分度：疑似唱歌区 sustained=21% vs 聊天区 6%。段边界 ±8s 设 ignore(-1) 不参与 loss。
3. **目标主播 A = 东雪莲**（数据集首位；管线与主播无关，标注到达后可即时切换）。
   A 的伪唱歌段=正例；A 的其余=负例。
4. **其他 3 位主播的全部音频 = 负例**（"有人在唱但不是A"），其中他们的伪唱歌段作为
   focus 区域过采样 → 直接逼模型学声纹差异，对应干扰 (d)。
5. **合成难负例**（stem 重混）：A 的伴奏 + 其他主播人声（响度匹配）= 干扰(d)完美合成；
   A 的伴奏单独 = 干扰(a)(c)；合成正例：A 人声 + 随机伴奏（增强鲁棒性，对应"伴奏不匹配仍是A在唱"）。

## 2. 模型路径（3 条并行对比）

共享前端：16kHz mono → log-mel(80, hop 320, 50fps)，fp16 缓存。输出 3.125fps（0.32s/帧）二分类「A 在唱」。

| | 路径 A：MelCRNN | 路径 A'：MelTCN | 路径 B：PANNs-头 |
|---|---|---|---|
| 结构 | 4×ConvBlock(时/频各16×池化)→BiGRU(2×256) | 同CNN→9级膨胀TCN(感受野±6min) | 冻结CNN14_16k嵌入(2048d@1.56fps)→BiGRU头 |
| 参数 | 2.1M | 3.9M | 头 ~1.7M（骨干80M冻结） |
| 依据 | 生产VAD共识形态 | MS-TCN长上下文共识，无循环全并行 | AudioSet预训练+轻头共识 |
| 风险 | 上下文窗内有限 | 边界平滑过度 | 窄带域差损伤嵌入 |

训练：window 96s、batch 32、AdamW 3e-4、cosine+warmup、bf16、BCE(带 ignore mask)、
SpecAugment+增益抖动；正例窗占比 50%；按 VOD 划分 train/val。
选型指标 = 0.5×frame-F1 + 0.5×event-F1（IoU≥0.5，真值与预测同享 90s 间隙合并语义）。

## 3. 后处理（产品语义层）

probs(3.125Hz) → 3s 中值平滑 → 双阈值滞回(on 0.6/off 0.4) → 去 <8s 噪点 →
填 ≤45s 间隙（间奏/短插放） → 合并 ≤90s 邻段（"唱-放别人对照-再唱"=一段，按需求定义） →
弃 <50s 段 → 前后 padding 5s → ffmpeg -c copy 快剪。
全部常数集中在 PostProcessConfig，依 val 调优。

## 4. 推理预算

PANNs 缓存实测 RTF≈1300×；mel+CRNN 更快。瓶颈是 ffmpeg 解码（≈30-60×RT）。
整链路（解码+特征+模型+切片）预计 RTF < 0.05，远优于"不超过直播时长"的要求；
显存 <3GB（10GB 约束轻松满足）。路径 B 若胜出，CNN14 骨干推理也仅 ~2GB bf16。

## 6. 已知风险与对策

- 伪标签噪声（A 聊天带 BGM 误标为唱）→ 段级 sustained/密度过滤 + ignore 缓冲；
  训练后用模型置信度反查伪标签离群段（self-training 一轮，时间允许的话）。
- 8kHz 窄带 → 全管线统一 16k 重采样训练=推理同分布；真实推理输入若是全频带，
  对 mel 高频段做随机带宽截断增强（TODO，视时间）。
